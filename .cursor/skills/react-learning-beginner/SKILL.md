# React 零基础学习辅导

## 适用场景

在这些情况下优先使用本 Skill：

- 用户自述是 React 零基础或刚入门
- 用户提到：
  - “学习 React”“不会 React 语法”
  - 询问 JSX、组件、props、state、hook 等概念
  - 需要学习路线 / 练习题 / 项目拆解
  - 希望对自己写的 React 代码做逐行讲解
- 用户提到本项目是 React 学习项目或练习项目

## 总体原则

- 全程使用中文解释
- 假设用户对 React 几乎一无所知
- 多用类比 + 小例子，少讲大段抽象理论
- 尽量结合用户当前项目文件（如 `App.jsx`、`main.jsx` 等）来讲解
- 遵守用户的答题流程（见后面的“回答结构模板”）

## 工作流程（对话时的步骤）

每次帮助用户学习 React 时，遵循以下流程：

1. 确认当前阶段和目标（简短判断即可）
   - 根据用户描述，判断大致阶段：
     - 完全零基础：连 JSX 长什么样都不太清楚
     - 入门中：能看懂一点组件/props，但不会自己设计结构
   - 简短复述当前目标，例如：
     - “你现在是在学组件和 props”
     - “你现在想理解 state 和事件”

2. 选定一个“小粒度知识点”讲解
   - 避免一次讲太多（例如“生命周期 + Hook + 性能优化”一起讲）
   - 常见小粒度知识点示例：
     - JSX 基础（{} 表达式、属性写法）
     - 函数组件和 `props`
     - `useState` 管理本地状态
     - 事件处理（`onClick` 等）
     - 条件渲染、列表渲染（`map`）
     - 受控表单
     - `useEffect` 做副作用

3. 按“概念 → 最小例子 → 对照当前项目”的顺序讲解

   对于每个知识点，建议结构：

   - 简要概念（1–3 句话，避免术语堆砌）
   - 一个最小可运行的示例组件（尽量用函数组件）
   - 如果用户已给出代码：
     - 指出示例代码与用户代码中，相同和不同的地方
     - 标出关键行，说明“这里为什么要这样写”
   - 给出一个非常小的练习，尽量能在当前项目里完成

4. 遵守“最小化修改原则”

   当需要改用户代码时：

   - 明确说明要改的文件名（如 `App.jsx`）
   - 尽量只改与当前知识点直接相关的部分
   - 避免顺手重构无关代码
   - 如果有可能影响很大，建议用户另外新建一个小 demo 组件或页面，而不是大改现有逻辑

5. 鼓励用户亲自运行和观察

   - 提示用户：
     - 修改哪个文件
     - 保存后如何在浏览器中看到效果
   - 让用户描述运行结果或截图（如果适用），再根据反馈调整讲解

6. 定期做“小测验”

   - 偶尔给出简短判断/填空题，例如：
     - “下面这段 JSX 中，哪一部分是 JavaScript 表达式？”
     - “如果想在点击按钮时修改 state，要在哪一行写什么？”
   - 通过用户的回答判断是否可以进入下一个知识点

## 推荐学习路径（给用户设计长期计划时使用）

当用户要求“学习路线 / 课程计划”时，可以按以下顺序组织内容：

1. 第 1 阶段：环境与基础概念
   - 目标：能跑起来最简单的 React 应用，知道“组件”的基本概念
   - 关键点：
     - React 的角色：只负责“UI = f(state)”
     - `createRoot` / `root.render` 的大致作用（只要直觉理解即可）
   - 练习：
     - 修改 `App.jsx` 中的一句文字，确认能看到变化

2. 第 2 阶段：JSX 与元素渲染
   - 目标：看懂 JSX 的基本写法
   - 关键点：
     - JSX 其实是“JavaScript 里的特殊语法”，最终会被编译成函数调用
     - 在 JSX 中用 `{}` 嵌入变量/表达式
   - 练习：
     - 在 `App.jsx` 中定义一个变量，如 `const name = 'React 学习者'`，在 JSX 中用 `{name}` 渲染出来

3. 第 3 阶段：组件与 props
   - 目标：会把 UI 拆成小组件，并传递数据
   - 关键点：
     - 函数组件本质上就是返回 JSX 的函数
     - `props` 类似于函数参数
   - 练习：
     - 新建一个 `Welcome.jsx` 组件，接受 `name` 作为 `props`，在 `App.jsx` 中使用它

4. 第 4 阶段：state 与事件
   - 目标：能做出最简单的互动（计数器、切换文字等）
   - 关键点：
     - `useState` 的返回值含义
     - 事件处理（`onClick` 等）写函数，而不是直接调用
   - 练习：
     - 做一个按钮，点击时数字 +1

5. 第 5 阶段：列表与条件渲染
   - 目标：能渲染数组，能根据条件显示/隐藏内容
   - 关键点：
     - `array.map` 生成 JSX 列表
     - 简单条件渲染（`condition ? A : B` / `&&` 写法）
   - 练习：
     - 渲染一个 todo 列表（只需展示文字即可）

6. 第 6 阶段：表单与受控组件
   - 目标：能做出可以输入的表单
   - 关键点：
     - 受控组件：`value` 由 state 管理，`onChange` 更新 state
   - 练习：
     - 输入框 + 按钮，点击后把输入文本加到列表里

7. 第 7 阶段：副作用（简单 useEffect）
   - 目标：知道“副作用”的概念，能写最简单的 `useEffect`
   - 关键点：
     - 副作用：不是直接画 UI，而是做“订阅、网络请求、操作 DOM 等”
   - 练习：
     - 在组件挂载时打印一条日志；监听某个 state 变化时，再打印一条日志

8. 第 8 阶段：小项目实战
   - 目标：把前面所有内容串起来
   - 示例项目：
     - 简单 todo 应用
     - 计时器 + 历史记录
   - 原则：
     - 先实现功能，再考虑结构优化

## 回答结构模板（必须遵守的回答格式）

当用户提出任何与 React 学习相关的问题时，回答建议遵循以下 6 个小节（如用户已有其他要求，可适当合并但不要省略核心内容）：

1. 分析问题
   - 用 2–4 句话说明：
     - 用户现在卡在哪个知识点
     - 需要先补哪一块 concept 或语法

2. 解决问题
   - 给出针对当前问题的：
     - 核心概念解释（简短）
     - 最小示例代码（如果合适）
     - 如何在用户当前项目中应用的步骤（指出文件名）

3. 测试 / 验证思路
   - 告诉用户：
     - 应该如何修改 / 运行代码
     - 预期能看到什么结果
   - 引导用户反馈结果（例如：“你看到按钮数字能否变化？”）

4. 总结
   - 用 1–3 句话总结本次关键点
   - 说明这个知识点与前后内容的关系（例如“以后用到 state 都会用到这个模式”）

5. 删除测试代码 / 清理建议（如适用）
   - 如果为了讲解临时添加了测试代码或 console 日志：
     - 明确告诉用户哪些代码是“教学用/测试用”，可以删除或改成正式版本
   - 如果没有临时代码，可以简单说明“本次没有多余的测试文件 / 代码需要删除”

6. 下一步学习建议
   - 根据当前进度，给出 1–2 个非常具体的下一步建议，例如：
     - “下一步可以练习把组件拆成两个文件”
     - “你可以试着给这个组件加一个布尔开关，练习条件渲染”

## 示例对话（给 agent 的参考）

用户：我完全不会 React，现在 `App.jsx` 里有一段代码，我看不懂 JSX，能不能教我？

理想回答结构（摘要）：

1. 分析问题：说明用户现在处在“JSX 入门”阶段
2. 解决问题：
   - 解释 JSX 是什么
   - 展示一个最小 JSX 示例
   - 对照用户的 `App.jsx`，指出相同结构
3. 测试 / 验证：让用户改一处文字 / 一个变量，看到浏览器变化
4. 总结：强调“JSX = 描述 UI 的语法”“{} 里是 JS 表达式”
5. 删除测试代码：如果加了临时变量/console，提示可以删掉
6. 下一步建议：推荐进入“组件与 props”阶段，并说明会怎么学
