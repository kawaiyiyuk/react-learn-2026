## Day 2：状态、列表与表单（约 2 小时）

### 目标
- 用 `useState` 管理待办列表
- 实现增、删、切换完成状态
- 筛选：全部 / 未完成 / 已完成
- 新增待办用受控表单

### 2.1 数据模型
在 `src` 下新建 `data/tasks.js`（或直接在 HomePage 里定义）：
```javascript
// 每条任务的结构
// { id: string, title: string, completed: boolean, note?: string }
```

### 2.2 首页状态设计（HomePage.jsx）
- `tasks`：数组，元素为 `{ id, title, completed, note? }`
- `filter`：'all' | 'active' | 'completed'
- `inputValue`：新增待办输入框的值（受控）

### 2.3 需要写的功能
1. **展示列表**  
   - 根据 `filter` 过滤后 map 渲染，每个 item 用 `task.id` 作为 `key`  
   - 显示 checkbox（是否完成）+ 标题，点击标题用 `Link to={/task/${task.id}}` 进详情  

2. **新增待办**  
   - 受控 input：value={inputValue} onChange 更新 inputValue  
   - 提交时：生成新 id（如 `crypto.randomUUID()` 或 `Date.now().toString()`），往 tasks 里 push `{ id, title: inputValue, completed: false }`，清空 inputValue  

3. **切换完成状态**  
   - 列表项上的 checkbox：onChange 时找到对应 task，把 `completed` 取反，用新数组 setTasks（不要直接改原数组）  

4. **删除**  
   - 列表项旁加删除按钮，点击后 filter 掉该 id，setTasks 新数组  

5. **筛选**  
   - 三个按钮「全部 / 未完成 / 已完成」，点击 setFilter  
   - 列表用 `tasks.filter(t => filter === 'all' || (filter === 'active' && !t.completed) || (filter === 'completed' && t.completed))`  

### 2.4 状态放在哪？
- 先把 `tasks` 放在 **HomePage**。详情页需要的「单条任务」先通过 **路由 id + 从列表里 find 这条** 得到（列表数据用 props 或后面 Day 3 的持久化读到同一份数据）。  

### 2.5 验收
- 能新增待办并显示在列表  
- 能勾选/取消完成  
- 能删除  
- 筛选三个状态切换正确

## Day 3：持久化与详情页笔记（约 2 小时）

### 目标
- 用 localStorage 持久化任务列表
- 详情页展示一条任务并可编辑「笔记」
- 用 `useEffect` 读/写 localStorage

### 3.1 自定义 Hook：useLocalStorage
在 `src/hooks/useLocalStorage.js` 中实现：

```javascript
import { useState, useEffect } from 'react'

export function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key)
      return item ? JSON.parse(item) : initialValue
    } catch {
      return initialValue
    }
  })

  useEffect(() => {
    try {
      window.localStorage.setItem(key, JSON.stringify(value))
    } catch (e) {
      console.error(e)
    }
  }, [key, value])

  return [value, setValue]
}
```

- 首页把 `const [tasks, setTasks] = useState([])` 改成 `const [tasks, setTasks] = useLocalStorage('tasks', [])`  
- 这样刷新页面后列表仍在  

### 3.2 详情页数据从哪来？
- 方案 A：把 `tasks` 和 `setTasks` 通过 **Context** 提升（Day 4 会做 Context，这里可先不提升）  
- 方案 B（简单）：详情页也用 `useLocalStorage('tasks', [])` 读同一份数据，根据 `useParams().id` 找到当前任务；编辑笔记后，用 setTasks 更新对应项（找到 id 那条，替换成 { ...task, note: newNote }），这样首页和详情页共享同一份持久化数据  

### 3.3 详情页（TaskDetailPage.jsx）要做的
1. `const { id } = useParams()`  
2. `const [tasks, setTasks] = useLocalStorage('tasks', [])`  
3. `const task = tasks.find(t => t.id === id)`  
4. 若 `!task` 则显示「未找到该任务」并 return  
5. 展示：标题、完成状态、笔记区域  
6. 笔记：一个受控的 `<textarea value={task.note ?? ''} onChange={...}>`，onChange 时 setTasks(更新这条的 note)  
7. 可选：删除按钮，删除后 `useNavigate()` 跳回 `/`  

### 3.4 验收
- 首页新增的待办，刷新后还在  
- 点进某条详情，能看到标题和笔记  
- 在详情页改笔记，刷新或回首页再点进来，笔记已保存  
- 若在详情页删除，能跳回首页且列表已更新

## Day 4：Context 主题与 useRef（约 2 小时）

### 目标
- 用 Context 管理主题（亮/暗）
- 设置页切换主题，布局和页面消费主题
- 首页新增待办后，用 useRef 让输入框重新 focus

### 4.1 ThemeContext
1. 新建 `src/context/ThemeContext.jsx`  
2. `createContext` 默认值如 `'light'`  
3. Provider 的 value 为 `[theme, setTheme]`，用 useState 管理  
4. 导出 `useTheme`：`useContext(ThemeContext)`  

### 4.2 挂载 Provider
在 `App.jsx` 里，用 `ThemeProvider` 包住 `Routes`（在 BrowserRouter 内部），这样所有页面都能用 `useTheme()`  

### 4.3 消费主题
- 在 `Layout.jsx` 或 `index.css` 中根据 theme 加 class 或 data 属性，例如 `<div className={theme}>` 或 `data-theme={theme}`，在 CSS 里写 `.dark { background: #1a1a1a; color: #eee; }` 等  
- 设置页（SettingsPage）：两个按钮「浅色 / 深色」，点击调用 setTheme('light') / setTheme('dark')  

### 4.4 持久化主题（可选）
- 用 `useLocalStorage('theme', 'light')` 替代 useState，这样刷新后主题保持  

### 4.5 useRef：提交后 focus
- 首页新增待办的表单：给 input 加 `ref={inputRef}`  
- `const inputRef = useRef(null)`  
- 提交成功后执行 `inputRef.current?.focus()`  

### 4.6 验收
- 在设置页切换主题，整站样式跟着变  
- 新增待办提交后，光标回到输入框
